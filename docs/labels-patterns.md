# Label patterns (modular)

This document defines the label naming patterns to use in the repository.

## Pattern rules

- `priority:<level>`
- `type:<kind>`
- `area:<domain>`
- `status:<state>`

## Allowed labels

### Priority
- `priority:low`
- `priority:medium`
- `priority:high`

### Type
- `type:feature`
- `type:bug`
- `type:chore`
- `type:docs`
- `type:test`
- `type:polish`

### Area
- `area:player`
- `area:runner`
- `area:collectables`
- `area:speed-system`
- `area:scenario`
- `area:ui`
- `area:audio`
- `area:build`
- `area:qa`
- `area:repo`

### Status
- `status:blocked`
- `status:needs-review`
- `status:ready-for-qa`
